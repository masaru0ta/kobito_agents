# タスク管理ルール

このプロジェクトでは `tasks/` ディレクトリでタスクを管理している。以下のルールに従うこと。

## タスクファイル形式

ファイル名: `task_{YYYYMMDDHHmmss}_{ランダム英数3文字}.md`

例: `task_20260404_143000_x7k.md`

```markdown
---
title: "タスクのタイトル"
agent: system
phase: draft
created: YYYY-MM-DD
---

## ゴール

このタスクで達成すること。

## 作業ステップ

- [ ] ステップ1
- [ ] ステップ2
- [ ] ステップ3
```

## 起票ルール

タスクファイルを作成する前に、必ず以下をユーザーと確認すること:
1. **タスクの概要** — 何をゴールとするか
2. **作業ステップ** — ゴールに至るまでのステップ分解

合意を得てからタスクファイルを起票する。推測で作業内容を書かない。

## 作業ルール

- 作業完了したステップは `- [x]` に更新する
- 全ステップ完了時に frontmatter の `phase` を `done` に更新する

## AI間通信

`ask_agent` ツールを使うと、他のエージェントに質問できる。

- `ask_agent(agent_id, message)` — 新規会話を開始
- `ask_agent(agent_id, message, session_id)` — 既存会話を継続

返却される `session_id` を保持すれば、同じ相手と継続的に会話できる。
自分自身に送信することはできない。
