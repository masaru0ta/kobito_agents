# cli_bridge.py リファクタリング

## 概要
プロジェクト全体で最低スコア（64/100）の `cli_bridge.py` の可読性・メンテナンス性・堅牢性を向上させる。

## 作業ステップ

- [x] 【作業前評価】`cli_bridge.py` 単体を精査し、問題箇所を網羅的にリストアップする
- [x] 推論判定ロジックの重複排除 — `inferring_session_ids` と `process_debug_info` が個別に実装している判定ロジックを共通関数に統合
- [x] PIDファイル管理の分離 — `_pid_dir`, `_write_pid_file`, `_remove_pid_file`, `cleanup_orphaned_processes` を独立モジュールに切り出し
- [x] デバッグprintのlogger統一 — `print(f"[DEBUG]...")` 等を適切なログレベルの `logger` 呼び出しに置換
- [x] テスト修正・追加 — importパス更新、抽出した共通関数の単体テスト追加
- [x] 【作業後評価】同じ基準で再評価し、改善度合いを確認する
