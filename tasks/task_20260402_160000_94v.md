## 背景

`app.js` が1459行の単一ファイルに肥大化している。全てグローバルスコープの変数・関数で構成され、関心の分離がない。Phase 2でタスク管理UIが追加されたことでさらに拡大しており、今後の機能追加に支障をきたす。

### 現状の構造

| セクション | 行数 | 主な問題 |
|-----------|------|---------|
| 状態変数（グローバル） | ~16行 | 12個のグローバル変数が散在 |
| エージェント一覧 | ~40行 | — |
| プロセスポーリング | ~100行 | `pollProcessStatus()` が再起動検知・ファイル監視・UI更新を兼務 |
| セッション一覧 | ~70行 | — |
| チャット履歴 | ~100行 | — |
| メッセージ送信 | ~180行 | `sendMessage()` がSSE解析・DOM操作・状態管理・エラーハンドリングを全て担当 |
| DOMキャッシュ | ~20行 | — |
| タブ・設定・リサイズ | ~120行 | — |
| タスク管理 | ~460行 | `renderTaskList()` 115行、`renderTaskDetail()` 105行 |

## 方針

ES Modules（`<script type="module">`）で分割する。ビルドツールは導入しない。

## 作業ステップ

- [ ] 共有状態を `state.js` に集約（グローバル変数 → exportされたオブジェクト）
- [ ] ユーティリティを `utils.js` に分離（`escapeHtml`, `showToast`, `formatTaskDate`）
- [ ] エージェント関連を `agents.js` に分離（`loadAgents`, `renderAgents`, `selectAgent`）
- [ ] セッション関連を `sessions.js` に分離（`loadSessions`, `renderSessions`, `selectSession`）
- [ ] チャット関連を `chat.js` に分離（`sendMessage`, `renderMessages`, `appendMessage`, `appendThinking` 等）
- [ ] タスク管理を `tasks.js` に分離（`loadTasks`, `renderTaskList`, `renderTaskDetail`, D&D 等）
- [ ] UI初期化を `ui.js` に分離（`initTabs`, `initResize`, `initActions`, `initResponsive`）
- [ ] `app.js` をエントリポイントに縮小（import + DOMContentLoaded のみ）
- [ ] 全機能の動作確認（セッション切替、チャット送受信、タスク操作、D&D）
