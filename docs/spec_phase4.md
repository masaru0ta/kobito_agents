# Phase 4 仕様書: エージェント追加・削除

## 概要

Web UIからエージェント（既存プロジェクトディレクトリ）を新規登録・削除できるようにする。
追加されたエージェントは既存のセッション・タスク・スケジューラー機能をそのまま利用できる。

## 設計原則

- **既存ディレクトリ前提** — プロジェクトディレクトリの新規作成は行わない。既に存在するディレクトリを指定して登録する
- **最小限のフォーム** — 必須項目のみ入力させ、残りはデフォルト値で補完する
- **systemエージェントの保護** — systemエージェントは削除不可とする

## バックエンド

### API

#### `POST /api/agents` — エージェント新規登録

リクエストボディ:

```json
{
  "name": "キャスパー",
  "path": "D:/AI/code/craftgame5",
  "description": "ゲームデザイナー",
  "cli": "claude",
  "model_tier": "deep"
}
```

処理:
1. バリデーション（後述）
2. `name` からIDを自動生成（日本語名の場合はローマ字変換せず、タイムスタンプベースのIDを生成）
3. `agents.json` に追記
4. 登録されたエージェント情報を返却

レスポンス: `200 OK` + エージェント情報

#### `DELETE /api/agents/{agent_id}` — エージェント登録解除

処理:
1. `agent_id` が `"system"` なら `403 Forbidden`
2. `agents.json` から該当エントリを除外
3. プロジェクトディレクトリ・セッションデータ等は一切削除しない（登録解除のみ）

レスポンス: `200 OK`

### バリデーション

| チェック | エラー時 |
|----------|----------|
| `name` が空でないこと | 400 |
| `path` が空でないこと | 400 |
| `path` が実在するディレクトリであること | 400 |
| `cli` が `"claude"` または `"codex"` であること | 400 |
| `model_tier` が `"deep"` または `"quick"` であること | 400 |
| 同一 `path` のエージェントが既に登録されていないこと | 409 |

### ID生成ルール

`agent_{YYYYMMDDHHmmss}_{ランダム3文字}` 形式とする。

例: `agent_20260404_143000_x7k`

理由: 名前からのスラッグ化は日本語名で破綻する。タスクIDと同じタイムスタンプ方式で統一する。

### ConfigManager 変更

`config.py` に以下のメソッドを追加:

- `add_agent(name, path, description, cli, model_tier) -> AgentInfo` — バリデーション + ID生成 + 永続化
- `delete_agent(agent_id) -> None` — systemエージェント保護 + 永続化

## フロントエンド

### エージェント追加UI

サイドバーのエージェント一覧セクションに「＋」ボタンを配置する。

```
エージェント [＋]
┌─────────────────┐
│ ● レプリカ       │
│   システム管理    │
└─────────────────┘
```

「＋」ボタン押下で、中央ペインにエージェント追加フォームを表示する。

### 追加フォーム

中央ペインを一時的にフォーム表示に切り替える。

| 項目 | 入力形式 | 必須 | デフォルト |
|------|----------|------|-----------|
| 名前 | テキスト | ○ | — |
| プロジェクトパス | テキスト | ○ | — |
| 説明 | テキスト | — | 空文字 |
| CLI | セレクト（claude / codex） | — | claude |
| モデル | セレクト（深い思考 / 軽い即答） | — | deep |

フォーム下部に「登録」「キャンセル」ボタン。

- 「登録」→ `POST /api/agents` → 成功時にエージェント一覧を再読み込みし、追加されたエージェントを選択状態にする
- 「キャンセル」→ フォームを閉じて元の表示に戻る

### エージェント削除UI

設定タブ内に「エージェント登録解除」ボタンを追加する。

- systemエージェント選択時は非表示
- 押下時に確認ダイアログ（「エージェント "○○" の登録を解除しますか？プロジェクトデータは削除されません。」）
- 確認後 `DELETE /api/agents/{id}` → 成功時にエージェント一覧を再読み込みし、先頭エージェントを選択

## 既存機能との連携

追加されたエージェントは以下が自動的に利用可能:

- **セッション** — `path` 配下の Claude Code セッションを SessionReader が読み取る
- **タスク** — `tasks/` ディレクトリからタスクファイルを TaskManager が読み取る（`agent` フィールドでフィルタ）
- **設定** — `path/CLAUDE.md` を読み書き（存在しない場合は空表示、保存時に新規作成）
- **スケジューラー** — 承認済みタスクがあればスケジューラーの実行対象になる

追加実装は不要。既存の仕組みがそのまま機能する。
