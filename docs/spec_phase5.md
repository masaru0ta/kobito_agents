# Phase 5 仕様書: ファイルブラウザ・セッション連携

## 概要

エージェントの作業ディレクトリ内のファイルを閲覧できる「ファイル」タブを実装し、
ファイル・タスクとチャットセッションを双方向に連携する仕組みを構築する。

---

## 1. ファイルタブ（ディレクトリブラウザ）

### 概要

旧「レポート」タブを「ファイル」タブに刷新。エージェントの作業ディレクトリを
ディレクトリ構成ごと掘り下げて閲覧できるリスト形式のブラウザを提供する。

### UI構成

```
[ファイルタブ]
  ┌─ ブレッドクラム（パス階層） ─────────── [名前順 | 更新順] ┐
  │                                                              │
  │  📁 src                                  4/4土 16:30 更新  │
  │     src/server/routes                                        │
  │                                                              │
  │  📄 CLAUDE.md                                               │
  │     このシステムの...          1.2KB · 4/5日 09:00 更新    │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘
```

#### ファイルエントリ（2行表示）

| 行 | 内容 |
|----|------|
| 1行目 | アイコン + ファイル名 |
| 2行目（ファイルのみ） | プレビューテキスト（日本語優先）+ サイズ + 更新日時 |
| 2行目（フォルダ） | 最終更新日時（配下ファイルの最大mtime） |

#### ファイルアイコン色分け

| 種別 | アイコン | 色 |
|------|---------|-----|
| フォルダ | 📁 | 黄 |
| Markdown | 📄 | 青 |
| HTML | 🌐 | 緑 |
| 画像 | 🖼️ | オレンジ |
| JSON | `{ }` | 黄 |
| ソースコード | `</>` | 紫 |
| その他 | 🔒 | グレー |

#### ソート

- **名前順**（デフォルト: 更新順）
- **更新順**: フォルダとファイルを `mtime` で混在ソート

### ファイル表示

| ファイル種別 | 動作 |
|-------------|------|
| Markdown | システム内でレンダリング（marked.js）|
| 画像 | システム内でインライン表示 |
| JSON | システム内でシンタックスハイライト表示 |
| ソースコード | システム内で highlight.js によるハイライト表示 |
| HTML | ブラウザの別タブで開く |

### プレビューテキスト生成ルール

1. 日本語文字（Unicode \u3000–\u9fff、\uff00–\uffef）を含む行を優先的に使用
2. 該当行がない場合は意味のある最初の行を使用
3. スキップするパターン: `---`（YAML frontmatter区切り）、JSDocブロック（`/**` ～ `*/`）、`#!`（シバン）、`//`・`#`・`*` で始まるコメント行

### 日時表示フォーマット

全ページ共通の `formatDate()` 関数で統一。

| 条件 | フォーマット |
|------|-------------|
| 当年 | `4/5日 09:00` |
| 過去年 | `2025/4/5日 09:00` |

## 2. バックエンド API

### `GET /api/agents/{agent_id}/reports`

ディレクトリ一覧を返す。

クエリパラメータ: `path` （省略時: ルート）

レスポンス:

```json
{
  "dirs": [
    { "name": "src", "path": "src", "mtime": 1743840000.0 }
  ],
  "files": [
    {
      "name": "CLAUDE.md",
      "path": "CLAUDE.md",
      "size": 1234,
      "mtime": 1743840000.0,
      "is_md": true,
      "is_image": false,
      "is_json": false,
      "is_code": false,
      "preview": "このシステムの設計者であり管理者である。"
    }
  ]
}
```

フォルダの `mtime` は配下ファイルの最大 mtime（浅い1階層スキャン）。

### `GET /api/agents/{agent_id}/reports/{filepath:path}`

ファイル内容を返す。

| ファイル種別 | レスポンス |
|-------------|-----------|
| Markdown / JSON / ソースコード | `PlainTextResponse` |
| 画像 / HTML | `FileResponse` |

## 3. ファイル ↔ セッション連携

### 概要

ファイル閲覧中に「ファイルについて話す」操作でそのファイルに紐づくチャットセッションを開始・再開できる。
ファイルとセッションは双方向にリンクされ、それぞれの画面からもう一方へ移動できる。

### ファイル詳細メニュー

ファイル詳細画面の右上に `⋮` メニューボタンを設置。

| メニュー項目 | 動作 |
|-------------|------|
| ファイルについて話す | 紐づきセッションを開く（なければ新規作成） |

#### 「ファイルについて話す」の動作フロー

```
既存セッションあり → セッションを選択してチャットタブへ
既存セッションなし → 新規チャット画面を開く（pendingFileLinkPath に保存）
                      ↓ 最初のメッセージ送信後にセッションIDが確定
                      ↓ POST /api/agents/{id}/file-links でリンクを保存
```

新規セッションは送信後に初めてIDが確定するため、
送信前にリンクを登録するのではなく、送信完了後に登録する（遅延登録パターン）。

### ファイル詳細のセッションバナー

ファイルに紐づくセッションが存在する場合、ファイル詳細上部にセッション情報を表示する。

```
[セッションタイトル] → クリックでチャットタブ + セッションを開く
```

### データ構造

#### ファイルリンク (`.kobito/file-links/{safe_path}.json`)

```json
{ "file_path": "tasks/task_xxx.md", "session_id": "uuid" }
```

`safe_path` はファイルパスの `/` `\` `:` を `__` に置換した文字列。

#### セッションメタ (`.kobito/meta/{session_id}.json`)

```json
{
  "linked_file": "tasks/task_xxx.md",
  "title": "ファイル名",
  "created": "2026-04-05T00:00:00+00:00"
}
```

### API

#### `GET /api/agents/{agent_id}/file-links?path={file_path}`

ファイルに紐づくセッションIDを返す。

```json
{ "session_id": "uuid" }
// または
{ "session_id": null }
```

#### `POST /api/agents/{agent_id}/file-links`

ファイルとセッションを紐づける。セッションメタにも `linked_file` を書き込む。

リクエスト:
```json
{ "file_path": "tasks/task_xxx.md", "session_id": "uuid", "title": "task_xxx.md" }
```

## 4. タスク ↔ セッション連携（逆リンク）

### 概要

タスクから相談・作業セッションを開始する際、セッションメタにタスクIDとタイトルの逆リンクを書き込む。
これにより、チャット画面からタスク詳細へ戻れるリンクを表示できる。

### データ構造

タスクメタ (`.kobito/tasks/{task_id}.json`) は従来通り:

```json
{
  "task_id": "task_xxx",
  "sessions": ["uuid1", "uuid2"],
  "talk_session_id": "uuid3"
}
```

セッションメタ (`.kobito/meta/{session_id}.json`) に逆リンクを追加:

```json
{
  "linked_task": "task_20260405_120000_abc",
  "linked_task_title": "Phase5 ファイルブラウザ実装"
}
```

### API 変更

#### `POST /api/agents/{agent_id}/tasks/{task_id}/sessions`

作業セッションIDをタスクに紐づける。加えてセッションメタに `linked_task` を書き込む。

#### `PUT /api/agents/{agent_id}/tasks/{task_id}/talk-session`

相談セッションIDをタスクに紐づける。加えてセッションメタに `linked_task` を書き込む。

## 5. チャットヘッダーの連携リンク

### 概要

チャットセッションにファイルまたはタスクが紐づいている場合、
チャットヘッダーの2行目にリンクを表示し、クリックで対象画面へ移動できる。

### 表示レイアウト

```
┌────────────────────────────────────────────────────┐
│ セッションタイトル                   [モデル選択]   │
│ 📎 CLAUDE.md  または  📋 タスクタイトル             │
└────────────────────────────────────────────────────┘
```

### 動作仕様

| リンク種別 | アイコン | クリック時の動作 |
|-----------|---------|----------------|
| ファイルリンク | 📎 | ファイルタブへ切替 → ディレクトリ遷移 → ファイルを開く |
| タスクリンク | 📋 | タスクタブへ切替 → タスク詳細を開く |

### セッション読み込み時の処理

`loadSessionHistory()` 実行時に `GET /api/agents/{id}/sessions/{session_id}/meta` を呼び出し、
メタの内容に応じてヘッダーリンクを表示・非表示を制御する。

優先順位: `linked_task` > `linked_file`

### API

#### `GET /api/agents/{agent_id}/sessions/{session_id}/meta`

セッションメタ（`.kobito/meta/{session_id}.json`）を返す。ファイルが存在しない場合は `{}` を返す。
